1. Для чого потрібен шаблон проєктування MVC?

Шаблон проєктування MVC використовується для логічного розділення вебзастосунку на три незалежні компоненти:
Model - відповідає за дані та бізнес-логіку
View - відповідає за відображення інформації користувачу
Controller - приймає запити користувача та координує взаємодію між Model і View
Використання MVC підвищує підтримуваність коду, спрощує тестування та дозволяє змінювати інтерфейс без впливу на бізнес-логіку.

2. В чому полягають відмінності методів GET та POST протоколу HTTP?

Метод GET використовується для отримання даних із сервера. Параметри передаються через URL, що робить їх видимими користувачу. GET-запити можуть кешуватися та є ідемпотентними.
Метод POST використовується для надсилання даних на сервер. Дані передаються в тілі запиту, що дозволяє передавати великі обсяги інформації та підвищує безпеку. POST-запити не є ідемпотентними та зазвичай не кешуються.

3. Що таке сервлет?

Сервлет - це Java-клас, який виконується на сервері та призначений для обробки HTTP-запитів від клієнта
Сервлет приймає запит, обробляє його відповідно до бізнес-логіки та формує HTTP-відповідь
Сервлети працюють у спеціальному сервлет-контейнері (наприклад, Apache Tomcat), який керує їх життєвим циклом

4. Якими способами передаються дані HTML-форм з клієнта на сервер? В чому полягають їх переваги і недоліки?

Основні способи передачі даних це використання методів GET та POST.

GET:
Переваги: простота, можливість кешування, зручність для пошуку
Недоліки: обмеження довжини URL, відсутність безпеки

POST:
Переваги: відсутність обмежень на обсяг даних, краща безпека
Недоліки: неможливість кешування, складніше налагодження

5. Чи впливає регістр символів при передачі імен параметрів у HTTP-запитах до сервлетів?

Так, імена параметрів чутливі до регістру
Параметри username, UserName та USERNAME вважаються різними

Тому при отриманні параметрів у сервлеті необхідно точно відповідати регістру, який використовується у формі або URL.

6. Чи впливає регістр символів при передачі значень параметрів у HTTP-запитах до сервлетів?

Так, значення параметрів також є чутливими до регістру.
Наприклад, значення admin і Admin розглядаються як різні.

Це особливо важливо при реалізації авторизації, фільтрації даних та перевірки ролей користувача.

7. В чому полягає різниця між статичними HTML-сторінками та сторінками JSP?

Статичні HTML-сторінки містять фіксований контент, який не змінюється залежно від даних або користувача
JSP-сторінки (JavaServer Pages) дозволяють динамічно генерувати HTML-контент на сервері, використовуючи Java-код, EL та JSTL
JSP застосовуються для створення динамічних вебінтерфейсів

8. В чому полягає різниця між сервлетами та JSP-сторінками?

Сервлети орієнтовані на реалізацію бізнес-логіки та обробку запитів
JSP-сторінки орієнтовані на представлення даних користувачу
На практиці сервлети використовуються як контролери, а JSP - як view у шаблоні MVC

9. Як виглядає життєвий цикл JSP-сторінки? Що таке трансляція та компіляція JSP-сторінок?

Життєвий цикл JSP складається з таких етапів:
Трансляція JSP у сервлет
Компіляція сервлету
Завантаження класу
Виконання та обробка запитів

Трансляція - це перетворення JSP у Java-сервлет, а компіляція — створення .class файлу

10. Що таке скриптлети та директивні елементи?

Скриптлети (<% ... %>) - це фрагменти Java-коду всередині JSP-сторінки, які виконуються на сервері
Директиви (<%@ ... %>) визначають налаштування JSP, наприклад, імпорт класів або кодування сторінки
У сучасних застосунках використання скриптлетів не рекомендується

11. В чому полягає різниця між директивою <%@ include %> та елементом <jsp:include>?

<%@ include %> виконується на етапі компіляції, і включений код стає частиною JSP
<jsp:include> виконується під час обробки запиту, дозволяючи динамічно включати контент

12. Що таке JavaBean? Як їх використовувати у JSP-сторінках?

JavaBean це Java-клас з приватними полями, публічними геттерами та сеттерами та конструктором без параметрів.
У JSP JavaBean використовується для передачі даних між сервлетом та сторінкою представлення через EL.

13. В чому полягають переваги та недоліки використання EL та JSTL у порівнянні зі скриптлетами та JSP-виразами?

EL та JSTL:
підвищують читабельність коду
зменшують кількість Java-коду у JSP
знижують ризик помилок і XSS-атак

Скриптлети ускладнюють підтримку та порушують принципи MVC.

14. В чому полягають переваги/недоліки використання JSTL-елемента <c:out> у порівнянні з EL та <%= ... %>?

<c:out> автоматично екранує HTML-символи, що захищає від XSS-атак
EL-вирази та JSP-вирази не завжди забезпечують таку безпеку без додаткових налаштувань

15. Який час життя атрибутів на рівнях requestScope, sessionScope, applicationScope? Чи можуть одночасно існувати атрибути з однаковими іменами?

requestScope — один HTTP-запит
sessionScope — сесія користувача
applicationScope — весь час роботи застосунку

Атрибути з однаковими іменами можуть існувати одночасно на різних рівнях.

16. Що таке XSS-атака? Як зробити сайт стійким до таких атак?

XSS-атака це впровадження шкідливого JavaScript-коду у вебсторінку
Захист здійснюється шляхом екранування даних, використання c:out, EL, а також політик безпеки (CSP)

17. Поясніть різницю між операціями forward та redirect.

Forward - внутрішня передача запиту на сервері без зміни URL у браузері.
Redirect - перенаправлення клієнта з формуванням нового HTTP-запиту та зміною URL.
