## 🚀 Як запустити проєкт локально

**1. Підніміть базу даних**
Переконайтеся, що Docker увімкнений, і виконайте в корені проєкту:
```bash
docker-compose up -d
```

**2. Додайте драйвер БД у GlassFish (ОБОВ'ЯЗКОВО!)**
GlassFish не вміє сам підключатися до MS SQL Server, йому потрібен драйвер.
* **Звідки взяти:** Після першої збірки проєкту (Build) файл `mssql-jdbc-13.2.1.jre11.jar` з'явиться у папці вашого проєкту за шляхом:
  `target/sport-schedule-1.0-SNAPSHOT/WEB-INF/lib/`
  *(або просто завантажте цей jar-файл з інтернету)*.
* **Куди скопіювати:** Знайдіть папку, куди ви встановили сервер GlassFish на своєму ПК. Скопіюйте файл драйвера у цю підпапку:
  `[Ваша_папка_GlassFish]/glassfish/domains/domain1/lib/ext/`
  *(Після копіювання обов'язково перезапустіть GlassFish, якщо він працював).*

**3. Створіть конфіг підключення**
Скопіюйте шаблон конфігурації ресурсу для GlassFish (це наш аналог `.env`):
```bash
cp src/main/webapp/WEB-INF/glassfish-resources.example.xml src/main/webapp/WEB-INF/glassfish-resources.xml
```
Відкрийте створений файл `glassfish-resources.xml` і запишіть ваш реальний пароль від локальної БД. *(Git проігнорує цей файл)*.
