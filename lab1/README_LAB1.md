1. Що таке архітектура «клієнт-сервер»?

Архітектура, у якій клієнт надсилає запити, а сервер їх обробляє та повертає результат.


2. В чому полягає різниця між HTML, HTTP та HTTPS?

HTML - мова розмітки

HTTP - протокол передачі даних

HTTPS - захищена версія HTTP з шифруванням (TLS)


3. В чому полягають відмінності методів GET та POST протоколу HTTP?

GET - передає дані в URL, безпечний, ідемпотентний

POST - передає дані в тілі запиту, не ідемпотентний


4. Що таке сервлет?

Java-клас, що обробляє HTTP-запити та формує HTTP-відповіді на сервері


5. Які є переваги/недоліки у використанні CGI-програм у порівнянні з сервлетами?

CGI: просто використовувати, але низька продуктивність (створюється новий процес для кожного запиту)

Сервлети: швидші, повторно використовують потоки, складніші


6. Чим сервер Apache відрізняється від сервера Apache Tomcat?

Apache HTTP Server - це вебсервер для обробки HTTP-запитів та статичного контенту.

Apache Tomcat - це сервлет-контейнер, призначений для виконання Java-сервлетів і JSP.

Tomcat також може працювати як вебсервер.


7. Для чого потрібен клас HttpServlet? Де він знаходиться? (у якому пакеті? у якій бібліотеці?)

HttpServlet це базовий клас для створення сервлетів, що працюють з HTTP.

Він надає методи doGet, doPost, doPut тощо

Знаходиться у пакеті jakarta.servlet.http бібліотеки Jakarta Servlet API.


8. Якими способами передаються данні HTML-форм з клієнта на сервер? В чому полягають їх переваги і недоліки?

Дані можуть передаватися за допомогою методів GET або POST.

GET зручний для простих запитів, але має обмеження довжини.

POST дозволяє передавати великі обсяги даних та є безпечнішим


9. Що таке заголовки HTTP (заголовки HTTP) та куки (HTTP-cookie)?

Заголовки - метадані запиту/відповіді

Куки - збережені на клієнті дані для сесій


10. Чи впливає регістр символів при передачі імен параметрів у HTTP-запитах до сервлетів?

Так, імена параметрів чутливі до регістру


11. Чи впливає регістр символів при передачі значень параметрів у HTTP-запитах до сервлетів?

Так, значення параметрів також чутливі до регістру


12. Як і коли використовують методи setStatus, sendError та sendRedirect?

setStatus встановлює HTTP-код відповіді без зміни потоку

sendError повертає код помилки та повідомлення

sendRedirect перенаправляє клієнта на інший URL
